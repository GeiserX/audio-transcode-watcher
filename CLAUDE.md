# CLAUDE.md — audio-transcode-watcher

## Overview
Containerized service that watches a source folder and automatically transcodes audio files to multiple formats simultaneously. Supports FLAC, ALAC, AAC, MP3, Opus, and WAV with synced lyrics fetching.

## Tech Stack
- Python 3.14 (application logic)
- FFmpeg (audio transcoding)
- watchdog (filesystem monitoring)
- Docker (Python 3.14-slim + FFmpeg)
- uv (dependency management, via uv.lock)
- pytest (testing)
- Codecov (coverage)

## Development
```bash
# Install dependencies
uv sync

# Run tests
pytest

# Run locally
python -m audio_transcode_watcher

# Docker Compose
docker-compose up -d
```

## Architecture
- `src/audio_transcode_watcher/` — Main application package
- `config.example.yaml` — Reference configuration file
- `tools/` — Utility scripts (e.g., verification)
- `tests/` — Test suite
- `docs/` — Documentation and images

## Key Rules
- Docker image published to Docker Hub as `drumsergio/audio-transcoder` with semver tags
- Atomic writes only — never leave partial or corrupted output files
- Orphan cleanup removes outputs when source files are deleted
- Safety guards prevent accidental mass deletion if folders appear empty
- Configuration via YAML file, not environment variables
